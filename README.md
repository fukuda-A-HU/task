# タスク管理

GitHub Issuesを使用したタスク管理システムです。

## 使い方

### デイリータスク

日々のタスクを管理する場合は、**デイリータスク**テンプレートを使用してください。

1. Issues ページで「New Issue」をクリック
2. 「デイリータスク」テンプレートを選択
3. タスク一覧、優先度、日付を記入
4. Issueを作成

### プロジェクトタスク

プロジェクト単位でタスクを管理する場合は、**プロジェクトタスク**テンプレートを使用してください。

1. Issues ページで「New Issue」をクリック
2. 「プロジェクトタスク」テンプレートを選択
3. プロジェクト名、概要、メインタスク、子タスクを記入
4. Issueを作成

#### 子タスクの記入方法

子タスクは以下のように記入してください：

```markdown
## メインタスク1の子タスク
- [ ] 子タスク1-1
- [ ] 子タスク1-2

## メインタスク2の子タスク
- [ ] 子タスク2-1
- [ ] 子タスク2-2
```

チェックボックスは、タスクが完了したら `- [x]` に変更することで進捗を管理できます。

## ラベル

- `daily`: デイリータスク
- `project`: プロジェクトタスク
- `task`: タスク全般

## テンプレート

- `.github/ISSUE_TEMPLATE/daily-task.yml`: デイリータスク用テンプレート
- `.github/ISSUE_TEMPLATE/project-task.yml`: プロジェクトタスク用テンプレート
